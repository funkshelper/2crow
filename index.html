Merchant - Create Escrow Address - Initiate Transaction

A Merchant should use this tab to generate a 2Crow address that customers can use to initiate a purchase. Paste an empty temporary bitcoin address below for which you have the private key. When you press submit, a code will be generated which you can place on your website or send to your customer.

Share this 2crow code with your customer.

Customer - Begin a Purchase

A customer should start here to begin a purchase. Enter the merchant's 2crow address in the first field. Enter an empty bitcoin address for which you have the private key in the second field. When you press submit, a 2crow escrow address will be created. You need to send this address on to the merchant. You can now fund this address.

Fund this address and share it with the merchant.

Merchant - Ship and Prepare Transaction for Finalizing

At this point the merchant should have received the escrow address and verified that it is funded using a block explorer. Now you need to say where you want the money to wind up. Paste the 2crow step 2 code received from the customer in the first field, your private key from step 1 in the second field, and the final receiving bitcoin address in the third field. Send the resulting code to the customer, with the goods or separately.

! Wrong private key

Send this code to the customer along with the product.

Customer - Finalize Transaction

To finalize the transaction and release the funds to the merchant. In the first field place the 2crow code sent by the merchant. In the second field place your private key saved from bitcoin address used in step 2.

! Wrong private key

Sign buyers transaction

Use this to initiate a refund if the funds are still in escrow. If the funds are moved out of escrow already, refunds can be handled with direct transactions. Merchant enters here the 2crow code and their escrow private key.

Address Generator

If you need a new bitcoin address keypair, use this form. Click "Generate" and a private key and bitcoin address will be generated which could be used by the merchant or customer when needed.

New keypair will appear in the textarea.

How It Works

This site is completely client side, it will also run on an offline computer

An escrow account is created that is controlled by both the buyer and the seller.

The buyer is unable to take the funds until the customer finalizes the transaction.
